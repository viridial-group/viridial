# Internationalization (i18n) for Viridial

This document describes how the SaaS supports multiple languages for frontend and backend.

Frontend (shadcn-vue)
- Use Vue i18n plugin (vue-i18n). Add locale files under `frontend/web/src/locales/{en,fr,es,ar}.json`.
- Detect Accept-Language and provide language switcher in UI.
- Build: include locale files in each app. Use `process.env.DEFAULT_LOCALE` as fallback.

Backend (Java Spring)
- Externalize messages to `src/main/resources/messages/messages_{locale}.properties`.
- Configure `LocaleResolver` (AcceptHeaderLocaleResolver or CookieLocaleResolver).
- Use `MessageSource` to resolve messages in controllers and validation messages.

Search
- Index language field in Meilisearch documents (e.g., `language: "fr"`).
- When searching, pass `filter=language=fr` or boost multilingual fields.

Environment
- `.env` should include `DEFAULT_LOCALE` and `SUPPORTED_LOCALES`.

Notes
- Translate all user-facing strings, email templates, and validation messages.
- For SEO public pages, ensure `hreflang` tags are present.
