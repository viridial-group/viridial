# US-FE-01: Design System & Component Library (UX/UI)

## Status: Draft

### Story
En tant que designer UI/UX, je veux un Design System et une bibliothèque de composants réutilisables afin d'assurer cohérence visuelle, accessibilité et accélérer le développement frontend.

### Acceptance Criteria
- Palette de couleurs, typographie, spacing et tokens documentés.
- Composants atomiques: Button, Input, Select, Modal, Badge, Avatar, Tooltip, Notification.
- Composants composés: Card annonce, Listing card, Header/Nav, Footer, Formulaire de création d'annonce.
- Composants responsives et testés aux breakpoints mobile/tablette/desktop.
- Accessibilité: WCAG AA (contraste, focus visible, labels aria).
- Storybook disponible avec exemples et tests visuels.

**Priority:** P0
**Estimation:** 5

### Tasks
- [ ] Créer `ui/` package ou dossier `frontend/components` avec Storybook
- [ ] Définir tokens (colors, typography, spacing) et documentation
- [ ] Implémenter composants atomiques et composés dans Storybook
- [ ] Ajouter tests unitaires et visual regression pour composants critiques
- [ ] Rédiger guide d'accessibilité et checklist

### Definition of Done
- [ ] Storybook et guide de style publiés localement
- [ ] Composants importables et testés
- [ ] Documentation du design system ajoutée
